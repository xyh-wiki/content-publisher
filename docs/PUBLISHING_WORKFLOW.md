# 内容生成、审核与多平台发布流程

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档基线 | 2026-07-22 |
| 目标读者 | 内容编辑、审核人员、渠道管理员和运营人员 |
| 业务规则 | [FUNCTIONAL_SPEC.md](FUNCTIONAL_SPEC.md) |
| 接口规则 | [API_REFERENCE.md](API_REFERENCE.md) |

本文档说明如何使用当前管理后台完成内容生产和发布。页面、渠道或操作步骤变化时必须同步更新。

## 2. 总体流程

```text
选择内容来源
  → 等待生成任务
  → 检查来源与 AI 内容
  → 编辑主稿并保存版本
  → Admin 审核
  → 发布预检
  → API 发布或人工发布
  → 核对任务、发布记录和渠道覆盖
```

平台坚持“一份可信主稿，多个渠道派生版本”。渠道内容不会覆盖主稿。

## 3. 创建内容

入口：`/projects`。

### 3.1 Git 项目

适用于根据真实代码仓库生成项目介绍、快速开始、架构说明或最佳实践。

1. 选择“Git 项目”。
2. 输入允许列表中的 HTTPS Git 地址。
3. 可选输入分支；为空时使用远程默认分支。
4. 提交导入并进入任务详情。
5. 等待项目状态变为 `READY`。
6. 在项目详情检查名称、描述、分支、Commit、语言和 License。
7. 配置生成策略并提交文章任务。

如果地址包含凭据、查询参数、Fragment，或主机/DNS 不符合安全策略，系统会在入队前拒绝。

### 3.2 主题教程

适用于不依赖单一代码仓库的教程、知识指南、最佳实践、排障或概念说明。

1. 选择“主题教程”。
2. 填写主题、内容说明和目标读者。
3. 选择文章类型和知识级别。
4. 填写关键词和可选参考说明。
5. 配置语言、语气、长度、禁用词和必备章节。
6. 提交并等待 `GENERATE_TOPIC_ARTICLE` 完成。

参考说明只是事实材料，不应包含需要模型执行的角色或操作指令。

### 3.3 网站推荐

适用于介绍一个公开网站、工具或服务的能力和适用场景。

1. 选择“网站推荐”。
2. 输入公开 HTTPS URL。
3. 填写推荐角度、目标读者和关键词。
4. 配置文章策略并提交。
5. 等待网站抓取和 `GENERATE_WEBSITE_ARTICLE` 完成。

网站必须可公开访问。回环、私网、危险重定向、响应过大或有效文字过少都会被拒绝。

## 4. 生成策略

三类来源共同支持：

- 输出语言和语气。
- 正文最小/最大长度，范围 200–3000。
- 关键词上限，范围 1–30。
- 禁用关键词。
- 必须出现的二级章节。

Git 来源可以单独填写必选关键词；主题和网站来源使用来源关键词作为必选词。

生成完成后先检查：

- 来源信息是否正确。
- 标题和摘要是否准确。
- 主关键词是否自然出现。
- 正文是否基于真实事实。
- H2/H3 是否清楚。
- 命令、版本、链接、兼容性和 License 是否可验证。
- 英文稿是否完整且不是机械翻译。

SEO 评分只提供编辑提示，不能代替事实审核。

## 5. 编辑与版本

入口：`/content` 或 `/articles/{articleId}`。

1. 打开 `DRAFT` 或 `REJECTED` 文章。
2. 修改中文标题、摘要、正文、标签和关键词。
3. 按需要修改英文稿。
4. 保存。
5. 确认版本号递增。

每次保存创建不可变版本。多人同时编辑时，基于旧版本的请求会被拒绝；刷新后重新核对再提交。

可以从历史版本恢复内容。恢复操作创建新版本，不删除或覆盖原历史版本。

## 6. 审核

只有 Admin 可以审核。

### 6.1 审核通过

审核前至少确认：

- 来源和事实正确。
- 不包含秘密、客户数据或未经授权代码。
- 标题、摘要和正文一致。
- 示例命令和版本真实。
- 标签、关键词和链接合理。
- 不包含虚构统计、评分、作者、客户、案例或许可证。
- 计划发布国外渠道时英文稿完整。

通过后文章进入 `APPROVED`，内容基线锁定。

### 6.2 驳回

驳回必须填写具体原因和下一步，例如：

- “项目版本与当前 README 不一致，请核对 Commit。”
- “英文摘要缺少使用场景，请补充后重新提交。”
- “示例命令无法从仓库验证，请删除或提供依据。”

驳回后文章进入 `REJECTED`，Editor/Admin 可以继续编辑并再次送审。

## 7. 发布预检

入口：`/publishing/articles/{articleId}`。

系统检查：

- 文章状态是否允许发布。
- 账号是否启用。
- 最近连接验证是否失败。
- 渠道是否有可运行适配器。
- 自托管地址是否安全。
- 凭据字段是否完整。
- 国外渠道是否有完整英文稿。
- canonical URL 和计划时间是否合法。

页面禁用不合格账号，并显示原因。直接调用 API 也会执行相同检查。

## 8. API 发布

### 8.1 可新接入渠道

- DEV
- WordPress
- Discourse
- GitHub Discussions
- Twitter/X
- Reddit
- Hashnode
- Mastodon
- Ghost

Medium 只支持已有合法 Integration Token 的存量账号，不能新建。

### 8.2 提交

1. 选择一个或多个已通过预检的账号。
2. 可选填写 HTTPS canonical URL。
3. 可选选择 UTC 计划时间，最远一年。
4. 提交。
5. 单渠道创建一个任务；批量发布为每个账号创建独立任务并共享批次 ID。

单次批量最多 20 个去重账号。

### 8.3 执行

- 计划时间到达前保持待执行。
- 工作器领取后进入运行中。
- X/Reddit 发布前自动刷新 Access Token。
- 成功后保存外部内容 ID 和 URL。
- 失败后保存有限错误，不暴露凭据。

外部 API 结果不确定时不会自动盲目重试。

### 8.4 取消和失败重试

- 未领取的 `PENDING` 或 `RETRY_WAIT` 可以取消。
- `RUNNING`、`SUCCEEDED`、`FAILED`、`CANCELLED` 不能取消。
- `FAILED` 发布可以由 Editor/Admin 在任务详情创建人工重试任务。
- 重试沿用原文章、账号和 canonical URL，原失败任务与发布记录保留。

重试前应先到目标平台核对是否已经实际发布，避免网络超时后重复发布。

## 9. 人工发布

人工渠道共 17 个：

- 小红书、CSDN、掘金、知乎。
- 博客园、SegmentFault、V2EX、开源中国。
- LinkedIn、微信公众号、简书、今日头条、B 站专栏。
- 51CTO 博客、腾讯云、阿里云、华为云开发者社区。

流程：

1. 在文章发布页选择人工渠道。
2. 进入 `/articles/{articleId}/manual/{channelType}`。
3. 检查自动适配的标题、正文和标签。
4. 按平台要求继续调整派生内容。
5. 复制内容。
6. 打开官方创作入口。
7. 在第三方页面登录、排版并发布。
8. 回填公开 HTTPS URL。
9. 保存。

系统校验 URL 域名与所选平台匹配，并保存最终派生内容快照、操作者和时间。

平台不保存第三方用户名、密码、Cookie 或验证码，也不执行模拟登录。

## 10. 内容适配矩阵

| 格式 | 渠道 | 主要规则 |
|---|---|---|
| Markdown 长文 | DEV、WordPress、Discourse、GitHub Discussions、Reddit、Hashnode、Medium、Ghost、CSDN、掘金、博客园、SegmentFault、V2EX、开源中国、51CTO、腾讯云、阿里云、华为云 | 保留 Markdown；按渠道转换 JSON/HTML 或附加原文链接 |
| 普通文本 | 小红书、知乎、LinkedIn、微信公众号、简书、今日头条、B 站专栏 | 去除 Markdown；遵守平台字符限制 |
| 短帖 | Twitter/X、Mastodon | 标题、摘要、原文链接和少量标签；限制 280/500 字符 |

小红书额外限制标题为 20 个 Unicode 字符，并生成规范化话题标签。

## 11. 发布中心

入口：`/publishing`。

### 待发布

查看已审核但尚未覆盖目标渠道的内容。

### 执行批次

- 按 `batchId` 展示多渠道任务。
- 有活动任务时每 5 秒刷新。
- 可以取消尚未领取的单个子任务。

### 发布记录

- 合并 API 与人工发布记录。
- 展示方法、状态、账号/操作者、时间、外链和有限错误。
- 不展示渠道凭据或人工正文快照。

### 覆盖分析

- 按文章和渠道展示最新记录。
- 未发布渠道明确标记为空缺。
- 历史记录继续保留在统一时间线。

## 12. 任务与运行检查

入口：`/jobs`、`/jobs/{jobId}`、`/monitoring`。

任务详情用于查看：

- 类型和状态。
- 进度百分比、标签和说明。
- 尝试次数和计划时间。
- 批次 ID。
- 结果资源。
- 有限错误码和说明。

监控大屏用于查看当前租户：

- 项目和文章来源。
- 任务总量、活动和状态。
- 发布成功/失败。
- 渠道账号和渠道表现。

## 13. 回收站

只有 Admin 可以使用 `/recycle-bin`。

- 活动任务不能删除。
- 有关联活动任务的文章不能删除。
- 删除文章会同时隐藏关联任务和发布记录。
- 恢复文章会同时恢复关联记录。
- 当前没有永久清理操作。

## 14. 异常处理建议

| 情况 | 处理 |
|---|---|
| 项目导入失败 | 检查 URL、允许主机、仓库大小、分支和网络 |
| 网站生成失败 | 检查 HTTPS、公开可访问性、重定向和正文长度 |
| AI 输出被拒绝 | 缩小约束冲突，检查必选词、禁用词、章节和长度 |
| 文章版本冲突 | 刷新文章，重新核对后保存 |
| 渠道预检失败 | 按页面原因补英文稿、验证连接、启用账号或修正地址 |
| 发布任务失败 | 先核对第三方平台，再决定是否人工重试 |
| 任务不可取消 | 任务已被领取或已结束，刷新状态 |
| 记录不可删除 | 等待关联活动任务结束 |
| 外链被拒绝 | 使用公开 HTTPS URL，并确认域名属于对应平台 |

## 15. 发布完成检查

每次发布完成后：

1. 在任务详情确认状态。
2. 在发布记录打开外部 URL。
3. 检查标题、正文、链接和格式。
4. 检查发布矩阵是否更新。
5. 对失败渠道记录原因和后续处理。
6. 不要通过删除失败记录掩盖问题。

## 16. 文档同步

内容来源、审核、渠道、内容格式、页面路径、发布预检、任务状态或人工流程变化时，必须同步更新本文档、`FUNCTIONAL_SPEC.md`、`TECHNICAL_DEVELOPMENT.md` 和 README。
